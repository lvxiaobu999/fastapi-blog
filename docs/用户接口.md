# 用户 CRUD 接口

当前用户接口用于开发阶段的数据管理，不包含 JWT、登录状态或权限判断。因此所有接口暂时公开，不能以当前状态直接暴露到生产网络。

## 接口列表

| 方法 | 地址 | 成功状态 | 用途 |
| --- | --- | --- | --- |
| `POST` | `/api/users` | `201` | 创建用户 |
| `GET` | `/api/users` | `200` | 分页列出用户 |
| `GET` | `/api/users/{user_id}` | `200` | 获取用户详情 |
| `PATCH` | `/api/users/{user_id}` | `200` | 部分更新用户 |
| `DELETE` | `/api/users/{user_id}` | `204` | 删除用户 |

列表接口支持 `offset` 和 `limit`，其中 `limit` 的范围是 1 到 100。

## 创建用户

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "password123"
}
```

Service 会规范化邮箱并使用 Argon2 哈希密码。响应不会包含 `password` 或 `hashed_password`。用户名或邮箱重复时返回 `409 Conflict`，请求字段不合法时返回 `422 Unprocessable Entity`。

## 更新用户

更新接口使用 `PATCH`，只修改请求中实际提供的字段：

```json
{
  "nickname": "Alice",
  "image_file": "profile-id.webp"
}
```

可更新字段包括 `username`、`email`、`password`、`nickname` 和 `image_file`。传入 `image_file: null` 会恢复默认头像。`is_admin` 不接受客户端修改。

## 初始化数据库

首次使用接口前，在项目根目录执行：

```powershell
uv run alembic upgrade head
```

该命令读取当前环境的 `DATABASE_URL`。开发环境会迁移 SQLite，生产环境执行前必须确认 `ENV` 和 PostgreSQL 连接地址正确。
