# 环境配置教程

项目使用 `pydantic-settings` 管理环境配置，并通过 `ENV` 在开发和生产配置之间切换。

## 配置文件

项目根目录包含三个环境文件：

```text
.env                 # 所有环境共享的配置
.env.development     # 开发环境配置
.env.production      # 生产环境配置
```

公共配置写入 `.env`，仅在特定环境下不同的值写入对应的环境文件。例如：

```dotenv
# .env
PROJECT_TITLE=FastAPI Blog
```

```dotenv
# .env.development
ENV=development
PROJECT_TITLE=FastAPI Blog (Development)
```

```dotenv
# .env.production
ENV=production
PROJECT_TITLE=FastAPI Blog
```

不要把密码、密钥或生产数据库连接提交到版本控制。环境文件包含敏感信息时，应将其加入 `.gitignore`，并提供不含真实凭据的 `.env.example` 作为说明。

## 配置优先级

配置值按以下优先级读取，靠前的值覆盖靠后的值：

1. 操作系统环境变量
2. 当前环境文件，例如 `.env.development`
3. 公共 `.env`
4. `Settings` 中定义的默认值

环境文件中的变量名不区分大小写，但建议统一使用大写下划线形式。

## 启动开发环境

没有设置 `ENV` 时，项目默认加载开发环境：

```powershell
uv run fastapi dev app/main.py
```

也可以显式指定：

```powershell
$env:ENV = "development"
uv run fastapi dev app/main.py
```

## 启动生产环境

PowerShell 中先设置环境，再启动应用：

```powershell
$env:ENV = "production"
uv run fastapi run app/main.py
```

`$env:ENV` 只对当前 PowerShell 会话及其子进程生效。需要切回开发环境时，可以重新赋值或执行：

```powershell
Remove-Item Env:ENV
```

## 在代码中读取配置

配置模型位于 `app/core/config.py`，通过缓存的 `get_settings()` 获取：

```python
from app.core import get_settings

settings = get_settings()
print(settings.env)
print(settings.project_title)
```

应用入口已经使用 `PROJECT_TITLE` 设置 FastAPI 标题。启动后访问 `http://127.0.0.1:8000/docs`，页面标题会反映当前环境的配置。

## 新增配置项

例如需要新增 `DEBUG`：

1. 在合适的环境文件中添加 `DEBUG=true`。
2. 在 `Settings` 中声明字段及类型：

   ```python
   debug: bool = False
   ```

Pydantic 会将环境变量文本转换为声明的 Python 类型；无效的配置会在应用启动时直接报出校验错误。
