# 环境配置教程

项目使用 `pydantic-settings` 管理环境配置，并通过 `ENV` 在开发和生产配置之间切换。

## 配置文件

项目根目录包含三个环境文件：

```text
.env                 # 所有环境共享的配置
.env.development     # 开发环境配置
.env.production      # 生产环境配置
```

公共配置写入 `.env`，仅在特定环境下不同的值写入对应的环境文件。例如：

```dotenv
# .env
PROJECT_TITLE=FastAPI Blog
```

```dotenv
# .env.development
ENV=development
PROJECT_TITLE=FastAPI Blog (Development)
```

```dotenv
# .env.production
ENV=production
PROJECT_TITLE=FastAPI Blog
```

不要把密码、密钥或生产数据库连接提交到版本控制。环境文件包含敏感信息时，应将其加入 `.gitignore`，并提供不含真实凭据的 `.env.example` 作为说明。

## 配置优先级

配置值按以下优先级读取，靠前的值覆盖靠后的值：

1. 操作系统环境变量
2. 当前环境文件，例如 `.env.development`
3. 公共 `.env`
4. `Settings` 中定义的默认值

环境文件中的变量名不区分大小写，但建议统一使用大写下划线形式。

## 启动开发环境

没有设置 `ENV` 时，项目默认加载开发环境：

```powershell
uv run fastapi dev app/main.py
```

也可以显式指定：

```powershell
$env:ENV = "development"
uv run fastapi dev app/main.py
```

## 启动生产环境

PowerShell 中先设置环境，再启动应用：

```powershell
$env:ENV = "production"
uv run fastapi run app/main.py
```

`$env:ENV` 只对当前 PowerShell 会话及其子进程生效。需要切回开发环境时，可以重新赋值或执行：

```powershell
Remove-Item Env:ENV
```

## 在代码中读取配置

配置模型位于 `app/core/config.py`，通过缓存的 `get_settings()` 获取：

```python
from app.core import get_settings

settings = get_settings()
print(settings.env)
print(settings.project_title)
```

应用入口已经使用 `PROJECT_TITLE` 设置 FastAPI 标题。启动后访问 `http://127.0.0.1:8000/docs`，页面标题会反映当前环境的配置。

## 新增配置项

例如需要新增 `DEBUG`：

1. 在合适的环境文件中添加 `DEBUG=true`。
2. 在 `Settings` 中声明字段及类型：

   ```python
   debug: bool = False
   ```

Pydantic 会将环境变量文本转换为声明的 Python 类型；无效的配置会在应用启动时直接报出校验错误。

## 数据库配置

项目根据当前环境校验数据库类型：

- `development` 必须使用 SQLite。
- `production` 必须使用 PostgreSQL。

开发环境已经在 `.env.development` 中配置本地 SQLite：

```dotenv
DATABASE_URL=sqlite:///./fastapi_blog.db
```

首次建立数据库连接时，SQLite 文件会创建在项目根目录。该文件已被 `.gitignore` 忽略。

生产数据库地址包含账号和密码，不应写入仓库。启动生产环境前，通过部署平台的密钥管理功能或操作系统环境变量提供：

```powershell
$env:ENV = "production"
$env:DATABASE_URL = "postgresql+psycopg://<user>:<password>@<host>:5432/<database>"
uv run fastapi run app/main.py
```

上述尖括号内容是占位符，需要替换为实际配置。不要将真实连接地址粘贴到文档、日志或版本控制中。

SQLAlchemy 的 Engine、Session 工厂和 FastAPI 数据库依赖位于 `app/db/session.py`。路由需要数据库会话时使用：

```python
from typing import Annotated

from fastapi import Depends
from sqlalchemy.orm import Session

from app.db import get_db

DbSession = Annotated[Session, Depends(get_db)]
```

当前改动仅建立连接配置和会话基础设施，不会自动创建表。后续引入 SQLAlchemy Model 和 Alembic 后，应通过迁移创建及更新数据表。
